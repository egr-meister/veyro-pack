# Veyro Pack

**Veyro Pack** is an offline packing checklist app that helps you prepare for trips
using ready-made trip templates and editable packing lists. Create a trip, pick a
trip type, work through the recommended packing categories, mark items as packed,
track your progress, and keep a small "Buy Before Trip" list — all stored locally
on your device.

It is a practical, calm travel utility built with **React Native + Expo**,
**Android-focused**, **portrait only**, and works fully **offline / in airplane mode**.

---

## Features

- Create trips with a name, type, and optional destination and dates.
- Five trip types, each with a tailored starter checklist: **Weekend, Beach, Business, Camping, Family**.
- Six packing categories: **Clothes, Documents, Hygiene, Electronics, Medicine Kit, Food**.
- Mark items packed/unpacked, add custom items, rename custom items, delete custom items.
- Move items into **Buy Before Trip**, and restore them back to the checklist.
- A separate **Buy Before Trip** shopping list: add, mark bought, delete, and move a bought item into the packing list.
- Travel-specific progress: a **suitcase fill meter**, packing status strips, and per-category progress bars.
- **Duplicate** a previous trip (resets packed/bought states, keeps custom items, new id and timestamp).
- Settings: clear packed states, replay onboarding, reset all local data, privacy note.
- Confirmation dialogs before deleting a trip or resetting all data.

## Offline-first

Veyro Pack does everything on-device. There is no backend, no Firebase, no
external API, no cloud sync, and no account. Nothing you enter ever leaves your
phone.

### No internet / no permissions

The app requests **no runtime permissions** (no internet, location, camera,
microphone, contacts, storage/gallery, or notifications). The Android manifest
does **not** include the `INTERNET` permission — `INTERNET` and
`ACCESS_NETWORK_STATE` are explicitly listed under `blockedPermissions` in
`app.json` so they are stripped from the merged manifest.

### Airplane mode support

Because the app never touches the network, it launches and works identically in
airplane mode. You can plan and pack a trip with no connectivity at all.

### Local storage

All data is persisted with **AsyncStorage** only:

- `@veyro/trips` — every trip, including its packing items and buy items.
- `@veyro/settings` — onboarding flag and preferences.

The storage layer (`src/storage/storage.js`) is defensive by design:

1. Loaded data is merged with default values.
2. Empty storage returns safe empty defaults.
3. Missing fields on trips/items are filled in by normalizers.
4. Corrupted JSON falls back safely instead of crashing.
5. A trip with no items, an empty category, missing route params, or missing
   settings never crash the UI — guards and optional chaining are used throughout.

Data survives app restarts.

Each type generates a relevant starter checklist (see `src/data/templates.js`).

## Packing categories

`Clothes`, `Documents`, `Hygiene`, `Electronics`, `Medicine Kit`, `Food`.
Each category has a title, a lightweight text/emoji icon, its checklist items, a
packed/unpacked count, and per-category progress.

## Buy Before Trip

A small list of things to pick up before you leave. You can:

- add items manually (optionally tagged with a category),
- mark items as bought,
- delete items,
- move a bought item into the packing checklist,
- and the trip detail screen shows the buy-list count.

You can also move a packing item into Buy Before Trip and restore it later. The
list is stored locally with the trip.

## Duplicate trip

Duplicating a trip creates a brand new trip with copied categories and items,
all packed states reset to `false`, all bought states reset to `false`, custom
items kept, a new unique `id`, and a new `createdAt`. The default name is
`Copy of <original name>`, which you can rename before saving.

## Design

### App icon concept

A rounded square (deep navy) with a soft sand suitcase, a calm teal checkmark on
the suitcase, and a small sky-blue travel tag accent. Readable at small sizes.
See `assets/icon.png` and `assets/adaptive-icon.png` (generated by
`make_assets.py`).

### Splash screen concept

App name "Veyro Pack" with a centered suitcase-and-check mark on a calm light
sand background. No heavy image assets. See `assets/splash.png`.

### Visual style — "Veyro Soft Travel Board"

Calm, practical, organized, modern, travel-focused, light premium utility style.
Palette: warm off-white background, deep navy text, muted sky-blue accents, soft
sand cards, calm teal progress elements, light gray dividers. No neon, no
cartoon look, no heavy gradients, no 3D assets, no photo backgrounds.

### Layout uniqueness — "Suitcase Planner Board"

The app deliberately avoids the generic "title → subtitle → stats card → vertical
stack of large buttons" template. Instead:

- **Home** is a compact planning board: a header with the app name and a small
  suitcase symbol, a settings icon in the header (not a big bottom button), an
  overall packing dashboard with a suitcase fill meter, a dark "Next trip"
  boarding-pass card with a packing strip, and trips shown as boarding-pass-style
  cards. The create action is a floating button.
- **Trip detail** feels like opening a suitcase: a navy "suitcase interior"
  header with the fill meter, a "side pocket" entry to Buy Before Trip, and
  categories shown as packing compartments.
- Progress is travel-specific (suitcase fill meter, packing strips, category
  progress) rather than a copied stats card. No mascots, no rewards, no childish
  styling.

## Getting started

### How to scaffold with the Expo template

This repository already contains the source. If you ever want to recreate the
base, the project was started from the Expo template:

```bash
npx create-expo-app veyro-pack
```

### How to install dependencies (always via `npx expo install`)

Do not hand-edit dependency versions. Install/repair them through Expo so they
match your Expo SDK:

```bash
# install everything in package.json
npm install

# add Expo core modules as direct dependencies
npx expo install expo-asset expo-constants expo-font expo-modules-core expo-splash-screen expo-status-bar expo-build-properties

# local storage
npx expo install @react-native-async-storage/async-storage

# navigation
npx expo install @react-navigation/native @react-navigation/native-stack react-native-screens react-native-safe-area-context

# align everything to the current SDK
npx expo install --fix
```

Every imported package is a **direct dependency** in `package.json` — nothing
relies on transitive dependencies.

### How to run locally

```bash
npm install
npx expo start
# press "a" to open on an Android emulator/device, or scan the QR in Expo Go
```

### Regenerate icon / splash (optional)

```bash
python3 make_assets.py
```

## How to build Android

Native folders are generated on demand (Continuous Native Generation), so first
prebuild, then build with Gradle.

```bash
# 1) generate the android/ project
npx expo prebuild --platform android

# 2a) build a release APK
cd android && ./gradlew assembleRelease

# 2b) build a release AAB (for Google Play)
cd android && ./gradlew bundleRelease
```

Outputs:

- APK → `android/app/build/outputs/apk/release/app-release.apk`
- AAB → `android/app/build/outputs/bundle/release/app-release.aab`

For a **signed** local build, pass the signing properties (same ones CI uses):

```bash
cd android && ./gradlew assembleRelease \
  -PVEYRO_UPLOAD_STORE_FILE="/absolute/path/veyro-pack-release-key.p12" \
  -PVEYRO_UPLOAD_STORE_PASSWORD="********" \
  -PVEYRO_UPLOAD_KEY_ALIAS="veyro_pack_key" \
  -PVEYRO_UPLOAD_KEY_PASSWORD="********"
```

If no signing properties are supplied, the build falls back to the debug signing
config (handy for quick testing).

## Android signing

**Never commit the keystore or passwords to the repository.** They are kept in
GitHub Secrets and injected at build time.

### How to generate the PKCS12 keystore

```bash
keytool -genkeypair -v -storetype PKCS12 -keystore veyro-pack-release-key.p12 -alias veyro_pack_key -keyalg RSA -keysize 2048 -validity 10000
```

Use the **same password** for the keystore and the key.

### How to add GitHub Secrets

Encode the keystore to base64 and add the values under
**Settings → Secrets and variables → Actions**:

```bash
base64 -i veyro-pack-release-key.p12 | tr -d '\n' > keystore.base64.txt
# copy the contents into the ANDROID_KEYSTORE_BASE64 secret
```

| Secret name                 | Value                                      |
|-----------------------------|--------------------------------------------|
| `ANDROID_KEYSTORE_BASE64`   | base64 of `veyro-pack-release-key.p12`     |
| `ANDROID_KEYSTORE_PASSWORD` | the keystore password                      |
| `ANDROID_KEY_ALIAS`         | `veyro_pack_key`                           |
| `ANDROID_KEY_PASSWORD`      | the key password (same as keystore)        |

## GitHub Actions

`.github/workflows/android-build.yml` runs on push to `main` (and manual
dispatch). It:

1. Checks out the repo.
2. Installs Node.js 20 and Java 17.
3. Installs dependencies with `npm install`.
4. Runs `npx expo install --fix`.
5. Runs `npx expo-doctor` and `npx expo install --check` (non-blocking).
6. Installs Android SDK Platform 35 and Build Tools 35.0.0.
7. Decodes the keystore from `ANDROID_KEYSTORE_BASE64`.
8. Prebuilds the Android project.
9. Builds the signed release **APK**.
10. Builds the signed release **AAB**.
11. Uploads both as workflow artifacts.

CI is responsible for a fast, stable APK/AAB build and signing only. There is no
mandatory emulator smoke-test on the free runners — see the local verification
checklist below.

## Google Play compatibility

### Android API 35 notes

- `compileSdkVersion 35` and `targetSdkVersion 35` (set in `app.json` via
  `expo-build-properties`). The project does **not** target API 34.
- `minSdkVersion 24`, compatible with React Native 0.79.
- Build Tools `35.0.0`.

### 16 KB page size compatibility

The app uses the current Expo SDK / React Native version with the New
Architecture enabled, whose native libraries are built to support Android 15+
**16 KB memory page sizes**. No old/incompatible native libraries are added; the
only native data dependency is AsyncStorage. The final AAB therefore supports
16 KB page sizes.

The app avoids the common Play rejections: it targets API 35 (not 34) and is
built against 16 KB-compatible native libraries.

## Release optimization

Standard Android **R8 / ProGuard** only (see `proguard-rules.pro`); no risky
third-party obfuscation libraries.

Recommended sequence:

1. First verify a **non-minified** release works:

   ```gradle
   minifyEnabled false
   shrinkResources false
   ```

   (This is the default Expo release configuration.)

2. Then enable shrinking/minification:

   ```gradle
   minifyEnabled true
   shrinkResources true
   proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
   ```

   With Expo you can flip these on by adding to the `expo-build-properties`
   plugin in `app.json`:

   ```json
   ["expo-build-properties", { "android": {
     "enableProguardInReleaseBuilds": true,
     "enableShrinkResourcesInReleaseBuilds": true
   }}]
   ```

3. Re-test launch after enabling minify/shrink.

## Local launch verification

A green CI build is **not** proof that the app launches. Before release:

```bash
adb install android/app/build/outputs/apk/release/app-release.apk
adb logcat
```

Confirm there are **no** errors such as:

- `Cannot find native module`
- `Module has not been registered`
- `Invariant Violation`
- `theme.fonts.regular is undefined`
- AsyncStorage JSON parse crash
- missing route params crash

Manually test the full flow:

- first launch with empty storage (onboarding shows)
- create a trip
- mark items packed
- add a buy item
- duplicate a trip
- delete a trip
- reset app data
- relaunch the app (data persists / resets as expected)
- launch in airplane mode
